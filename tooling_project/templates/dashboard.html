Let's start creating a new project framework!


There are 3 things to do -
  1. Create a New Repo
  2. Get Sonarcloud connected to the repo
  3. Get Travis connected to the repo

First, Let's create the new repo

Create New Repo

Second, Let's get Sonarcloud installed!
  • Navigate to Sonarcloud and click on "Login" and choose, "Login With Github"
  • Follow the instructions to analyse your project
  • Once that "webhook" is completed, everytime you push code to your master/main branch, sonarcloud will analyse it!

Third, Let's get Travis installed!
  • This is similar to Sonarcloud, but requires a configuration file written in yaml
  • Navigate to Travis-CI and click on "Login" and choose, "Login With Github"
  • Follow the instructions to install the Travis-CI webhook
  • Then, follow the instructions Here to write the config file in your project
  • Click on this LINK to see the travis config file in this app's source code and model yours after it
  • What you want to do, is enter your student email address, and your lecturer's email address in the notifications section.
  • HINT - The Travis build will fail if the "script" attribute is not set. The "script" attribute sets the testing path for Travis, and if there are no tests, the build will fail. Copy the "script" attribute from the source code.

You're Done! Good luck in your development!
